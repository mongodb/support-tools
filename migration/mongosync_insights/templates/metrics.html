About the Embedded Verifier

The Embedded Verifier is mongosync's built-in verification that confirms data was transferred correctly by hashing documents on both source and destination clusters.

  • Can Write: True — The definitive signal that the embedded verifier has passed and found no mismatches. Until verification completes, Can Write remains False.
  • Phase — Shown for source and destination independently. stream hashing = actively verifying documents; idle = not yet started or between operations.
  • Lag Time Seconds — How far behind the verifier is in checking documents. High values mean verification takes longer to complete after commit.
